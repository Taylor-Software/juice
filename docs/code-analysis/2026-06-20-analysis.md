**Toolchain:** Dart/Flutter · `flutter_lints` + `strict-casts` + `strict-raw-types`

## Headline

Healthy codebase. `flutter analyze` is **clean (0 issues)**, **1026 tests pass**, overall line coverage **85.1%** (9029/10612). No critical bugs, no security issues, no data-loss risk — and three "crash" findings from the automated scan were verified as **false positives** (see Methodology). The real signal is concentrated in three places: (1) a repeated **undisposed-dialog-`TextEditingController`** pattern across ~7 sites, (2) a few **god-objects** that have grown past comfortable size (`models.dart` 2537, `journal_screen.dart` 2115, `providers.dart` 1539, `map_screen.dart` 1466), and (3) small **duplication** worth DRYing. Top-3 priorities: sweep the dialog-controller leaks, extract the hex-mutation boilerplate in `MapNotifier`, and break up `journal_screen.dart`'s 339-line `build()`.

## Findings

### 🔴 Critical
- None. (Analyzer clean; no data-loss; the scan's force-unwrap "crashes" are guarded — see Methodology.)

### 🟠 High
- **Undisposed dialog controllers (pattern, ~7 sites)** — `TextEditingController`s created inside `showDialog` builders are never disposed → a small native-memory leak per dialog open. Sites: [scenes_pane.dart:74](lib/features/scenes_pane.dart#L74), [battle_pane.dart:34](lib/features/battle_pane.dart#L34), [resources_pane.dart:33](lib/features/resources_pane.dart#L33), [encounter_screen.dart:355](lib/features/encounter_screen.dart#L355) (×3), [sheet_widgets.dart:393](lib/features/sheet_widgets.dart#L393) (`renameDialog`), [tracks_pane.dart:125](lib/features/tracks_pane.dart#L125), [journal_screen.dart:1493](lib/features/journal_screen.dart#L1493). **Fix:** one shared `Future<String?> promptText(...)` helper that owns + disposes the controller in a `finally`; replace the ad-hoc dialogs with it (also kills duplication).
- **`journal_screen.dart` `build()` is ~339 LOC** ([362-700](lib/features/journal_screen.dart#L362)) mixing filter chips, entry list, and composer; `_entry()` is another 127 LOC. **Fix:** extract `_filterBar()`, `_entryList()`, `_composerBar()` (latter already exists) into widget methods; pull `_entry` row pieces into small `StatelessWidget`s.

### 🟡 Medium
- **`MapNotifier` hex-mutation boilerplate** — ≥6 methods repeat "find hex by col/row → mutate → save" ([providers.dart:637-861](lib/state/providers.dart#L637)). **Fix:** a private `_updateHex(col,row, HexCell Function(HexCell))`; collapses the copy-paste.
- **`MapNotifier` god-object (27 public methods)** mixing dungeon + hexcrawl concerns ([providers.dart:637-962](lib/state/providers.dart#L637)). **Fix:** partition hexcrawl mutations into a `HexcrawlNotifier` or a `_HexOps` helper. (Judgment call — it's cohesive-ish; do it if it keeps growing.)
- **Duplicate `_fmt(int)`** signed-number formatter defined identically in [dnd_sheet.dart:8](lib/features/dnd_sheet.dart#L8) and [shadowdark_sheet.dart:8](lib/features/shadowdark_sheet.dart#L8). **Fix:** hoist to `sheet_widgets.dart` (same move as the shared `abilityBox` in #85).
- **Sheet `maybeFromJson` JSON-helper duplication** — `intOr`/`strOr`/`strSet` patterns re-inlined across `DndSheet`/`ShadowdarkSheet` ([models.dart:1306-1356, 1499-1529](lib/engine/models.dart#L1306)). **Fix:** top-level `_parseIntOr`/`_parseStrSet` helpers.
- **Painter recomputes min/max every paint** — `_DungeonPainter.paint` ([map_screen.dart:444](lib/features/map_screen.dart#L444)) and `roomIdAt` ([map_screen.dart:32](lib/features/map_screen.dart#L32)) each `rooms.map().reduce(min/max)`. Low real impact (maps are small, repaints infrequent), but trivial to pass `minX/minY` from `_canvas`. **Fix:** thread the already-computed bounds into the painter/hit-test.
- **`TextPainter` allocated per hex in `_HexPainter.paint`** ([map_screen.dart:1372-1452](lib/features/map_screen.dart#L1372)) — 1–3 per visible cell. Fine for current map sizes; revisit only if hexfields get large.
- **Guarded-but-fragile force-unwraps** — `_saveAsEntity` `_saveAsKind(entry)!` ([journal_screen.dart:1301](lib/features/journal_screen.dart#L1301)) and `parseSlash(text)!` ([journal_screen.dart:1151](lib/features/journal_screen.dart#L1151)) are reachable only on the non-null path today, but a refactor could break that. **Fix:** early-return on null instead of `!`.

### 🟢 Low
- **`_titleCase` vs `_titleize`** — two title-casers ([sheet_widgets.dart:358](lib/features/sheet_widgets.dart#L358) vs [tables_screen.dart:9](lib/features/tables_screen.dart#L9)). Consolidate if behavior matches.
- **`_onComposerChanged` mixes string-parse + setState** ([journal_screen.dart:86-108](lib/features/journal_screen.dart#L86)). Extract a pure `parseComposerState(text, selection)` → testable record.
- **`jsonDecode` per provider build** for persisted state ([providers.dart](lib/state/providers.dart) multiple). Minor; only matters if a provider rebuilds hot. Cache the decoded value in the notifier if profiling ever flags it.

### 💡 Ideas
- **`models.dart` is 2537 LOC** — cohesive (all domain models) but large. Consider splitting by domain (`sheets.dart`, `map_models.dart`, `encounter.dart`) for navigability. Not urgent.
- **Legacy `juice.log.v1` migration** ([providers.dart:71-92](lib/state/providers.dart#L71), `sessionScopedKeys`, `campaign_io.dart:71`) is ~30 LOC of one-shot v1→v2 journal migration. CLAUDE.md documents it as intentional; "pre-release, no back-compat" suggests it *could* be dropped — but only if no real pre-v2 data exists on your devices. **Owner call** — don't remove blind.
- **Tighten the linter** — current is `flutter_lints` + strict casts/raw-types. `very_good_analysis` would add null-safety/await lints that would have flagged the fragile `!`s above. Weigh against churn.
