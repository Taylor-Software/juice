# Code Analysis — 2026-07-04

**Project:** Solo Adventurer's Journal (`juice`)
**Scope:** 153 source files / 46,486 LOC, 248 test files / 33,314 LOC, 4 integration tests
**Toolchain:** Flutter/Dart · `flutter_lints` + `strict-casts` + `strict-raw-types` + 3 extra rules

## Headline

The codebase is in very good health: `flutter analyze` is clean, disposal hygiene is
near-perfect (68 State/ChangeNotifier classes scanned, 3 dialog-scoped leaks only),
painters diff correctly in `shouldRepaint`, there are no O(n²) hot paths, no secrets,
no dead code beyond one stray probe test, and every pubspec dependency is used.
The three priorities: (1) per-keystroke full-list persistence in the character sheets,
(2) per-keystroke uncached search scans (journal + campaign search), and (3) the
pure-engine rule is violated by five `lib/engine/*_data.dart` files importing
`flutter/services.dart`.

## Findings

### 🔴 Critical

- None found.

### 🟠 High

- **H1** `lib/state/providers.dart:69` + 24 `onChanged: (v) => _save(...)` sites across 8 sheet
  files (argosa, cairn, dnd, draw_steel, kal_arath, knave, nimble, ose) — every keystroke in a
  freeform sheet TextField re-encodes the **entire character list** to JSON and writes
  SharedPreferences. Fix: debounced text field (trailing ~400 ms + flush on dispose).
- **H2** `lib/engine/{oracle,emulator,verdant,lonelog,hexcrawl}_data.dart` — all five import
  `package:flutter/services.dart` (`rootBundle`), violating the "lib/engine/ = pure Dart" rule.
  Fix: move `rootBundle.loadString` into the providers (`lib/state/providers.dart` already
  imports it); engine classes keep their `XData(json)` constructors.
- **H3** `lib/features/journal_screen.dart:709` (input at `:918`) — `searchEntries` re-scans the
  visible journal (regex mention-stripping per entry) inside `build()` on every keystroke.
  Fix: debounce the search input.
- **H4** `lib/features/campaign_search_sheet.dart:61` (input at `:104`) — `searchCampaign`
  re-scans all five entity lists on every keystroke. Fix: debounce.
- **H5** `lib/features/tracker_screen.dart:321-467` — the identical 10-line sheet-dispatch block
  (`if (c.X != null) return XSheetView(... onBack: ...)`) is copy-pasted **14×**, one per sheet
  type (~120 duplicated LOC). Fix: single `_premadeSheetView(c, onBack)` helper.

### 🟡 Medium

- **M1** `lib/state/campaign_io.dart:84-131` — campaign import validates 11 of 16
  `sessionScopedKeys`; `juice.{verdant,context,decks,gmchat,light}.v1` are imported with **no
  validation**, and the catch-all error hides which key failed. Fix: validate all keys + name
  the key in the `FormatException`.
- **M2** `CLAUDE.md` — stack notes are stale: the cloud interpret seam
  (`lib/state/cloud_interpreter.dart`, `cloud_key_store.dart`; `anthropic_sdk_dart`, `http`,
  `flutter_secure_storage`) shipped but the docs still describe a no-network stack.
- **M3** `analysis_options.yaml` — only 3 extra lint rules. Add `unawaited_futures`,
  `cancel_subscriptions`, `close_sinks`, `cast_nullable_to_non_nullable`.
- **M4** Dialog-scoped `TextEditingController` leaks ×3: `lib/features/quick_ref_view.dart:148`,
  `lib/features/tasks_pane.dart:88`, `lib/features/custom_table_editor.dart:11-15`. Fix:
  post-frame dispose after the dialog resolves (the `rumors_pane.dart:93` pattern).
- **M5** `lib/engine/models.dart` — the clamp ceiling `1 << 20` is repeated **29×** uncommented.
  Fix: one named constant documenting "arbitrary large cap, not a rules value".
- **M6** `lib/state/blob_store_io.dart:31` — blob id is interpolated into a file path
  unsanitized. Ids are content-addressed today, so no live exploit, but a `/`- or `..`-bearing
  id from a future call site would escape the blobs dir. Fix: reject ids containing `/` or `..`.
- **M7** `test/zz_journal_starve_probe_test.dart` — untracked print-only probe test (51 LOC, no
  assertions). Delete.

### 💡 Ideas

- **I1** `Character` (models.dart) carries ~14 nullable sheet fields; a sealed sheet sum-type
  would centralize dispatch — worth it only if the sheet roster keeps growing.
- **I2** `CharacterNotifier` has 24 public methods; the 11 `addX()` one-liners could collapse
  into `addPreMadeSheet(systemKey)` at call sites.
- **I3** `_leadVitals` (tracker_screen.dart:1072) and `characterHpPool` (models.dart:3655) both
  dispatch on sheet type. Deliberately distinct (multi-bar + extras vs single pool) — left
  as-is; a new sheet system must touch both.
