# Maps (Redesign Phase 6) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A Maps tool (Exploration group) with two tabs: a dungeon map grown room-by-room from the dungeon-room oracle (drunkard's-walk placement, corridors, tap-for-detail, linger), and a wilderness hex map revealed by travel rolls (env-seeded hexes via the existing 2dF drift, Lost markers, manual reveal). Both persist per-session, travel in campaign files, and can drop text snapshots into the journal.

**Architecture:** Pure placement/geometry engine in `lib/engine/map_builder.dart` (deterministic under seeded `Dice`; rooms connected by construction; offset-coordinate hex neighbors). `MapState{rooms, corridors, currentRoomId, hexes, currentHex}` persisted by a `MapNotifier` at `juice.map.v1.<sessionId>` (CrawlNotifier shape). One `MapScreen` tool with Dungeon|Hex tabs; `CustomPainter` + `InteractiveViewer` rendering with hit-testing kept in pure, unit-testable functions. The map tool owns its own 'New room' / 'Travel' actions (calling the existing `Oracle.dungeonRoom`/`dungeonLinger`/`wildernessTravel` and, for travel, saving the shared `crawlProvider` state) — zero coupling into the generators screen; travel made from the map reveals hexes, travel made from the Exploration tool still just moves the crawl (documented limitation, fine for v1).

**Tech Stack:** Existing rails; no new packages. Spec: `docs/superpowers/specs/2026-06-11-journal-redesign-design.md` (Phase 6). Baseline: 177 tests green; analyze = 1 pre-existing info (models.dart dangling doc comment). Key existing APIs: `Oracle.dungeonRoom() → GenResult{title, rolls, summary?, asText}`, `Oracle.dungeonLinger() → GenResult`, `Oracle.wildernessTravel(CrawlState) → ({GenResult result, CrawlState state})` (state.envRow 1..10, state.lost), `oracle.data.table('wilderness_environment')` (10 names), `crawlProvider` (save/reset), house notifier patterns in providers.dart.

**Engine (`lib/engine/map_builder.dart`) — pure functions over models + `Dice`:**

```dart
/// Grid position for the next room: a free 4-neighbor cell of the current
/// room, direction picked by [dice]; if the current room is boxed in, walk
/// (deterministically, dice-tiebroken) along connected rooms until one has
/// a free neighbor. Returns the chosen position AND the room it attaches to.
/// First room (empty list) goes to (0,0) attached to nothing.
({int x, int y, String? attachTo}) nextRoomPosition(
    List<DungeonRoom> rooms, String? currentId, Dice dice);

/// Odd-q offset hex neighbors (6), as (col,row) pairs. Pure geometry.
List<({int col, int row})> hexNeighbors(int col, int row);

/// Position for the next revealed hex: a dice-picked neighbor of the
/// current hex that is not yet revealed; if all 6 are revealed, the
/// dice-picked neighbor anyway (travel re-enters a known hex — returns
/// that existing cell's coords). Empty field -> (0,0).
({int col, int row, bool alreadyRevealed}) nextHexPosition(
    List<HexCell> hexes, int? curCol, int? curRow, Dice dice);
```

Implementation notes (binding): direction order N,E,S,W (dungeon) / the 6 odd-q neighbors in fixed documented order (hex); `dice.dN(n)` picks among candidates so seeded runs are deterministic; the boxed-in walk = BFS over corridor-connected rooms from current, visiting in insertion order, first room with a free neighbor wins (dice picks among ITS free neighbors). Rooms are connected by construction: caller adds corridor `[attachTo, newId]` whenever attachTo != null.

**Notifier (providers.dart, CrawlNotifier shape, key 'juice.map.v1'):**

```dart
class MapNotifier extends AsyncNotifier<MapState> {
  // build/save/_scopedKey/_ready per house pattern
  Future<DungeonRoom> addRoom({required String title, required String detail, required Dice dice});
    // uses nextRoomPosition; appends room + corridor; sets currentRoomId; returns the new room
  Future<void> selectRoom(String id);            // currentRoomId (no-op if unknown)
  Future<void> appendRoomDetail(String id, String extra); // detail += '\n' + extra (linger)
  Future<HexCell> revealHex({required int envRow, required bool lost, required Dice dice});
    // nextHexPosition; if alreadyRevealed -> update that cell's lost flag + move current; else append; moves current
  Future<void> revealHexAt(int col, int row, int envRow);  // manual reveal (no current move); no-op if occupied
  Future<void> resetDungeon();                   // clears rooms/corridors/current, keeps hexes
  Future<void> resetHexes();                     // inverse
}
```
