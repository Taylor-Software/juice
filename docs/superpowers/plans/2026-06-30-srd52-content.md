# SRD 5.2 content (data) — PR1 of 2

Adds D&D SRD 5.2.1 spells + monsters to the content library, edition-tagged
`'5.2'`, MERGED alongside the existing 5.1 entries in the same asset files. This is
the **data** half; PR2 adds the per-campaign edition preference + Reference/picker
edition filter.

**License:** SRD 5.2.1 is CC-BY-4.0 (in scope — see [[licensing-constraint]]).
Source vendored under `data/dnd_srd_52/` (from github.com/springbov/dndsrd5.2_markdown,
itself CC-BY-4.0). Required attribution added to `kContentAttributions`.

## Existing shape (do not break)
- `assets/spells_dnd.json` — 319 entries, `edition: "5.1"`, ids `dnd-<slug>`.
- `assets/foes_dnd.json` — 334 entries, `edition: "5.1"`, ids `dnd-<slug>`.

## Task 1 — `build_dnd52_content.py` (source of truth; never hand-edit JSON)
Parses the vendored markdown and merges 5.2 into the asset files. Idempotent:
on each run, read the existing asset, DROP any `edition=="5.2"` entries, append the
freshly-parsed 5.2 set, write back (5.1 entries pass through untouched).

**Spells** — `data/dnd_srd_52/07_Spells.md`, section after `## Spell Descriptions`:
- each `#### <Name>` (strip surrounding `**`) starts a spell.
- next italic line `*Level N School (Class, Class)*` OR `*School Cantrip (Classes)*`
  → `level` (cantrip→0), `school`, `classes` (split `, `).
- `**Casting Time:** …`, `**Range:** …`, `**Components:** …`, `**Duration:** …`
  → matching fields; `concentration` = "Concentration" in duration; `ritual` =
  "Ritual" in casting time.
- body paragraphs until the next `####`/`###`/`##` → `description`.
- `**_Using a Higher-Level Spell Slot._**` / `**_Cantrip Upgrade._**` paragraph →
  `higherLevels`.
- `id: dnd-2024-<slug>`, `system: dnd`, `edition: "5.2"`.

**Monsters** — `data/dnd_srd_52/12_MonstersA-Z.md` + `13_Animals.md`:
- each `## <Name>` starts a monster (skip the file's `# ` H1 title).
- `*Size Type, Alignment*` → `size` (first word), `creatureType` (the type word(s)).
- `- **Armor Class:** N` → `ac`; `- **Hit Points:** N (…)` → `maxHp` = N;
  `- **Speed:** …` → `speed`.
- the `|STAT|SCORE|MOD|SAVE|` table → `abilities {STR,DEX,CON,INT,WIS,CHA}` (SCORE).
- `- **CR** X (…)` → `cr` (string; keep fractions `1/2`,`1/4`,`1/8`; `—`→ omit).
- `### Traits` block: each `***Name.*** text` → `StatTrait` in `traits`.
- `### Actions` (+ Bonus Actions / Reactions / Legendary Actions) `***Name.*** text`
  → `Attack {name, detail}` in `attacks` (prefix the section to the name for
  non-Actions, e.g. "Reaction: Parry", to keep them distinguishable; keep it simple).
- `id: dnd-2024-<slug>`, `edition: "5.2"`.

**Self-verify** (raise on failure, like the other build_*_content.py):
- both editions present after merge: spells with `5.1` ≥ 319 AND `5.2` ≥ 250;
  foes `5.1` ≥ 334 AND `5.2` ≥ 200 (tune to actual parsed counts; the repo has
  ~320 spell + ~235 monster headers — some headers are section dividers, so accept
  the real parsed count and assert a sane floor).
- all ids unique across the merged file; every 5.2 id starts `dnd-2024-`.
- every spell: `level` in 0..9, non-empty `name`/`description`.
- every monster: 6 ability keys, non-empty `name`, `maxHp` > 0.
- print a summary (counts per edition).

Run order (document in the script header): `python3 build_dnd_content.py` (5.1)
THEN `python3 build_dnd52_content.py` (adds 5.2). Copy nothing by hand.
